## 60-Second Setup

Get professional code formatting across 20+ languages in three simple steps:

<Steps>
  <Step title="Install pre-commit">
    ```bash
    pip install pre-commit
    ```
  </Step>

  <Step title="Add hook configuration">
    Create `.pre-commit-config.yaml` in your repository root:

    ```yaml .pre-commit-config.yaml
    repos:
      - repo: https://github.com/duolingo/pre-commit-hooks.git
        rev: 1.13.3
        hooks:
          - id: duolingo
    ```
  </Step>

  <Step title="Install and run">
    ```bash
    pre-commit install
    pre-commit run --all-files
    ```
  </Step>
</Steps>

That's it! Your code is now formatted according to Duolingo's standards.

## Complete Working Example

Here's a complete example showing the hook in action with all three available hooks:

<CodeGroup>
  ```yaml .pre-commit-config.yaml
  repos:
    - repo: https://github.com/duolingo/pre-commit-hooks.git
      rev: 1.13.3
      hooks:
        # Speed up Docker image pulls
        - id: precache-docker
        
        # Format code across all languages
        - id: duolingo
        
        # Sync AI coding rules (optional)
        - id: sync-ai-rules
  ```

  ```python Before: unformatted.py
  import sys
  import os
  from typing import Dict,List
  
  def  hello(  name : str  )->str:
      result=f"Hello {name}"  
      return result
  
  
  data =  dict()
  items = list()
  ```

  ```python After: formatted.py
  import os
  import sys
  from typing import Dict, List
  
  
  def hello(name: str) -> str:
      result = f"Hello {name}"
      return result
  
  
  data = {}
  items = []
  ```
</CodeGroup>

<Note>
  The hooks automatically formatted the code by:
  - Sorting imports alphabetically
  - Fixing spacing around operators and colons
  - Replacing `dict()` with `{}`
  - Replacing `list()` with `[]`
</Note>

## Language-Specific Examples

The hook automatically detects file types and applies the appropriate formatters:

<Tabs>
  <Tab title="Python">
    ```python
    # Before
    import sys
    import os
    from typing import Dict,List
    
    class  Example(object):
        def  method( self , x:int )->int:
            return  x*2
    
    # After
    import os
    import sys
    from typing import Dict, List
    
    
    class Example:
        def method(self, x: int) -> int:
            return x * 2
    ```

    Formatters applied: **Ruff**, **autoflake**, **sed**
  </Tab>

  <Tab title="Java">
    ```java
    // Before
    public class Example{
        public void method(String name,int age){
            System.out.println("Hello "+name);
        }
    }
    
    // After
    public class Example {
      public void method(String name, int age) {
        System.out.println("Hello " + name);
      }
    }
    ```

    Formatters applied: **google-java-format**, **sed**
  </Tab>

  <Tab title="Kotlin">
    ```kotlin
    // Before
    fun hello(name:String):String{
        val items=listOf()
        return"Hello $name"
    }
    
    // After
    fun hello(name: String): String {
      val items = emptyList()
      return "Hello $name"
    }
    ```

    Formatters applied: **ktfmt**, **sed** (replaces `listOf()` with `emptyList()`)
  </Tab>

  <Tab title="Go">
    ```go
    // Before
    package main
    import "fmt"
    func main(){fmt.Println("Hello")}
    
    // After
    package main
    
    import "fmt"
    
    func main() { fmt.Println("Hello") }
    ```

    Formatters applied: **gofmt**, **sed**
  </Tab>
</Tabs>

## Configuration Options

Customize the hook behavior with optional arguments:

<CodeGroup>
  ```yaml Python 2 Project
  repos:
    - repo: https://github.com/duolingo/pre-commit-hooks.git
      rev: 1.13.3
      hooks:
        - id: duolingo
          args:
            - --python-version=2
  ```

  ```yaml Scala 3 Project
  repos:
    - repo: https://github.com/duolingo/pre-commit-hooks.git
      rev: 1.13.3
      hooks:
        - id: duolingo
          args:
            - --scala-version=3
  ```

  ```yaml With Docker Precaching

        - id: precache-docker
  ```

</CodeGroup>

## Common Workflows

### Running on Specific Files

```bash
# Run on specific files
pre-commit run --files src/app.ts src/utils.ts

# Run on all Python files
pre-commit run --files **/*.py

# Run on all staged files
pre-commit run
```

### Running in CI

Add to your CI pipeline to enforce formatting:

<CodeGroup>
  ```yaml GitHub Actions
  name: Pre-commit
  
  on: [push, pull_request]
  
  jobs:
    pre-commit:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v3
        - uses: actions/setup-python@v4
          with:
            python-version: '3.11'
        - name: Run pre-commit
          run: |
            pip install pre-commit
            pre-commit run --all-files
  ```

  ```yaml GitLab CI
  pre-commit:
    image: python:3.11
    before_script:
      - pip install pre-commit
    script:
      - pre-commit run --all-files
  ```

  ```groovy Jenkins
  pipeline {
    agent any
    stages {
      stage('Pre-commit') {
        steps {
          sh '''
            pip install pre-commit
            pre-commit run --all-files
          '''
        }
      }
    }
  }
  ```
</CodeGroup>

### Updating Hook Version

Keep your hooks up to date:

```bash
# Update to latest version
pre-commit autoupdate

# Update specific hook
pre-commit autoupdate --repo https://github.com/duolingo/pre-commit-hooks.git
```

## What Gets Formatted?

The hook automatically formats files based on their extension:

| File Extensions | Tools Applied |
|----------------|---------------|
| `.js`, `.jsx` | Prettier, ESLint, sed |
| `.ts`, `.tsx` | Prettier, ESLint, sed |
| `.py` | Ruff, autoflake, sed (Python 3)<br/>Black, isort, autoflake, sed (Python 2) |
| `.java` | google-java-format, sed |
| `.kt`, `.kts` | ktfmt, sed |
| `.go` | gofmt, sed |
| `.scala`, `.sbt`, `.sc` | scalafmt, sed |
| `.sh`, `.bash`, `.zsh` | shfmt, sed |
| `.cpp`, `.proto` | ClangFormat, sed |
| `.tf` | terraform fmt, sed |
| `.pkr.hcl` | packer fmt, sed |
| `.svg` | SVGO, sed |
| `.toml` | Taplo, sed |
| `.xml` | xsltproc, sed |
| `.css`, `.scss` | Prettier |
| `.html` | Prettier |
| `.md`, `.markdown` | Prettier |
| `.yaml`, `.yml` | Prettier |

<Info>
  The `sed` transformation runs on all files and handles universal formatting like trailing whitespace removal and EOF newlines.
</Info>

## Performance Tips

<AccordionGroup>
  <Accordion title="Use precache-docker for faster runs">
    The `precache-docker` hook pulls Docker images once instead of multiple times in parallel:

    ```yaml
    hooks:
      - id: precache-docker  # Add this first
      - id: duolingo
    ```
  </Accordion>

  <Accordion title="Run on changed files only">
    Instead of `--all-files`, run on staged changes:

    ```bash
    pre-commit run
    ```
  </Accordion>

  <Accordion title="Skip hooks temporarily">
    When you need to commit without running hooks:

    ```bash
    git commit --no-verify
    ```

    <Warning>
      Use sparingly—you'll still need to fix formatting issues later!
    </Warning>
  </Accordion>
</AccordionGroup>
