## Welcome to Duolingo Pre-commit Hooks

A comprehensive collection of [pre-commit](https://pre-commit.com/) hooks that bring professional code formatting and optimization to your development workflow. These hooks run formatters and linters across 20+ languages, all configured to minimize friction by auto-fixing violations instead of requiring manual corrections.

Used in production by Duolingo across all repositories, these hooks ensure consistent code quality on developer workstations and in CI environments.

## Supported Languages & Tools

The main `duolingo` hook runs formatters and linters for:

- **JavaScript/TypeScript** - Prettier v3.5.3, ESLint v9.23.0
- **Python 3** - Ruff v0.7.3, autoflake v1.7.8
- **Python 2** - Black v21.12b0, isort v5.13.2
- **Java** - google-java-format v1.24.0
- **Kotlin** - ktfmt v0.53, gradle-dependencies-sorter v0.14
- **Go** - gofmt v1.23.3
- **Scala** - scalafmt v3.8.3
- **Shell** - shfmt v3.10.0
- **C++/Protobuf** - ClangFormat v18.1.8
- **Infrastructure** - terraform fmt v1.9.8, packer fmt v1.14.2
- **Markup** - SVGO v3.3.2 (SVG), Taplo v0.9.3 (TOML), xsltproc (XML)

<Note>
  All formatters are configured to auto-fix violations automatically. You won't need to manually correct formatting issues.
</Note>

## Design Philosophy

<Steps>
  <Step title="Auto-fix Everything">
    Only rules that can be automatically fixed are enabled. No rules requiring manual intervention are enforced.
  </Step>
  <Step title="Minimize Friction">
    Sensible defaults that work for most projects without configuration tweaking.
  </Step>
  <Step title="Production Ready">
    Battle-tested across all Duolingo repositories, running on developer workstations and in CI.
  </Step>
  <Step title="Performance First">
    Parallel execution and Docker image caching ensure fast feedback loops.
  </Step>
</Steps>

## Quick Start

Get up and running in under a minute:

```yaml .pre-commit-config.yaml
repos:
  - repo: https://github.com/duolingo/pre-commit-hooks.git
    rev: 1.13.3
    hooks:
      - id: duolingo
```

Then install and run:

```bash
pre-commit install
pre-commit run --all-files
```
