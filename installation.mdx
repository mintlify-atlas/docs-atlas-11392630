## Prerequisites

Before installing the hooks, ensure you have the following installed on your system:

<Steps>
  <Step title="Install pre-commit">
    The hooks require [pre-commit](https://pre-commit.com/) to be installed. Choose your preferred installation method:

    <CodeGroup>
      ```bash pip
      pip install pre-commit
      ```

      ```bash homebrew
      brew install pre-commit
      ```

      ```bash conda
      conda install -c conda-forge pre-commit
      ```
    </CodeGroup>

    Verify the installation:

    ```bash
    pre-commit --version
    ```
  </Step>

  <Step title="Install Docker">
    The main `duolingo` hook runs inside a Docker container to ensure consistent formatting across all environments.

    <CodeGroup>
      ```bash macOS
      # Install Docker Desktop from https://www.docker.com/products/docker-desktop
      # Or use Homebrew:
      brew install --cask docker
      ```

      ```bash ubuntu/debian
      # Install Docker Engine
      sudo apt-get update
      sudo apt-get install docker-ce docker-ce-cli containerd.io
      ```

      ```bash fedora/rhel
      # Install Docker Engine
      sudo dnf install docker-ce docker-ce-cli containerd.io
      ```
    </CodeGroup>

    Verify Docker is running:

    ```bash
    docker --version
    docker ps
    ```

    <Note>
      Make sure Docker is running before attempting to use the hooks. The container image `duolingo/pre-commit-hooks:1.13.3` will be pulled automatically on first use.
    </Note>
  </Step>
</Steps>

## Adding Hooks to Your Project

<Steps>
  <Step title="Create configuration file">
    If your project doesn't already have a `.pre-commit-config.yaml` file, create one in your repository root:

    ```bash
    touch .pre-commit-config.yaml
    ```
  </Step>

  <Step title="Add hook configuration">
    Add the Duolingo hooks to your `.pre-commit-config.yaml`. Here's a complete example with all three hooks:

    ```yaml .pre-commit-config.yaml
    repos:
      - repo: https://github.com/duolingo/pre-commit-hooks.git
        rev: 1.13.3
        hooks:
          # Optimization hook for Docker image caching
          - id: precache-docker
            args:
              - ubuntu:22.04  # Optional: add custom images to precache
          
          # Main code formatting hook
          - id: duolingo
            args:
              - --python-version=3  # Default is Python 3
              - --scala-version=2.12  # Default is Scala 2.12
          
          # AI rules synchronization (optional)
          - id: sync-ai-rules
    ```

    <Warning>
      Always declare `precache-docker` **before** any other `language: docker_image` hooks to benefit from the caching optimization.
    </Warning>
  </Step>

  <Step title="Configure hook arguments (optional)">
    The `duolingo` hook accepts optional arguments to customize behavior:

    | Argument | Description | Default | Example |
    |----------|-------------|---------|----------|
    | `--python-version` | Python version for your project | `3` | `--python-version=2` |
    | `--scala-version` | Scala version for your project | `2.12` | `--scala-version=3` |

    <Tip>
      For Python 2 projects, use `--python-version=2` to enable Black and isort instead of Ruff.
    </Tip>
  </Step>

  <Step title="Install the hooks">
    Run the installation command to set up Git hooks in your repository:

    ```bash
    pre-commit install
    ```

    You should see output like:

    ```
    pre-commit installed at .git/hooks/pre-commit
    ```
  </Step>

  <Step title="Run hooks on existing files">
    Run the hooks on all files in your repository to establish a baseline:

    ```bash
    pre-commit run --all-files
    ```

    <Note>
      The first run will take longer as Docker pulls the hook container image. Subsequent runs will be much faster.
    </Note>

    Expected output:

    ```
    Precache Docker images......................Passed
    Duolingo....................................Passed
    ```
  </Step>
</Steps>

## Excluding Files and Directories

The following files and directories are automatically excluded:

- `build/` - Build output directories
- `node_modules/` - Node.js dependencies
- `.claude/skills/.generated/` - Generated AI skill files
- `AGENTS.md` - Generated by sync-ai-rules hook
- `copilot-instructions.md` - Generated by sync-ai-rules hook
- `gradlew` - Gradle wrapper script

<Tip>
  You don't need to add these to your hook's `exclude` key—they're handled automatically.
</Tip>

To exclude additional files or patterns, add an `exclude` key to specific hooks:

```yaml .pre-commit-config.yaml
repos:
  - repo: https://github.com/duolingo/pre-commit-hooks.git
    rev: 1.13.3
    hooks:
      - id: duolingo
        exclude: ^(vendor/|legacy/old_code\.py)$
```

## Editor Integration

For the best developer experience, configure your editor to use the same formatting settings:

<Steps>
  <Step title="Copy EditorConfig">
    Copy or symlink the `.editorconfig` file from this repository to your home directory:

    ```bash
    # From the duolingo/pre-commit-hooks repository
    cp .editorconfig ~/
    # Or create a symlink
    ln -s "$(pwd)/.editorconfig" ~/
    ```
  </Step>

  <Step title="Install EditorConfig plugin">
    Most modern editors have built-in or plugin support for [EditorConfig](https://editorconfig.org/):

    - **VS Code**: Install the "EditorConfig for VS Code" extension
    - **IntelliJ/PyCharm**: Built-in support (no plugin needed)
    - **Sublime Text**: Install the "EditorConfig" package
    - **Vim**: Install the `editorconfig-vim` plugin
  </Step>
</Steps>

## Verification

Verify your installation is working correctly:

<CodeGroup>
  ```bash Test on a single file
  # Create a test file with formatting issues
  echo 'const  x=1;' > test.js
  
  # Run pre-commit on just this file
  pre-commit run --files test.js
  
  # Check that it was formatted
  cat test.js
  # Output: const x = 1;
  ```

  ```bash Test on all files
  # Run on all files in the repository
  pre-commit run --all-files
  ```

  ```bash Check Docker container
  # Verify the Docker image was pulled
  docker images | grep duolingo/pre-commit-hooks
  ```
</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Docker daemon not running">
    If you see an error like `Cannot connect to the Docker daemon`, ensure Docker is running:

    ```bash
    # macOS with Docker Desktop
    open -a Docker
    
    # Linux
    sudo systemctl start docker
    ```
  </Accordion>

  <Accordion title="Hooks not running on commit">
    If hooks don't run when you commit, ensure you ran `pre-commit install`:

    ```bash
    pre-commit install
    ```

    Verify the hook is installed:

    ```bash
    ls -la .git/hooks/pre-commit
    ```
  </Accordion>

  <Accordion title="Slow first run">
    The first run downloads the Docker image, which can take a few minutes depending on your connection:

    ```bash
    # Pre-pull the image manually
    docker pull duolingo/pre-commit-hooks:1.13.3
    ```
  </Accordion>

  <Accordion title="Permission denied errors">
    If you see permission errors with Docker, ensure your user is in the docker group:

    ```bash
    sudo usermod -aG docker $USER
    newgrp docker
    ```
  </Accordion>
</AccordionGroup>
